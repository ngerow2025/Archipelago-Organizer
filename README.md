# Archipelago Organiser

This project is a Qt-based organizer/launcher for the local A Dance of Fire and Ice + Archipelago setup. The source code lives in `src/`, while the rest of the workspace is mostly generated by builds, overlay mounts, or local game/runtime setup.

## Workspace Layout

| Folder | Purpose | Typical contents |
| --- | --- | --- |
| `Game/` | Local game install tree used as the lower overlay source. | `A Dance of Fire and Ice.exe`, `A Dance of Fire and Ice_Data/`, `Bundles/`, `MonoBleedingEdge/`, `UnityCrashHandler64.exe`, `UnityPlayer.dll`, `User/`, `steam_appid.txt` |
| `mod/ADOFAI_AP-Mod/` | Local mod deployment bundle copied into the game environment. | `BepInEx/`, `doorstop_config.ini`, `winhttp.dll` |
| `upper/` | Writable overlay layer for the game runtime. | `BepInEx/LogOutput.log`, `BepInEx/cache/`, `BepInEx/config/`, `BepInEx/patchers/` |
| `merge/` | Merge mount point for the overlay filesystem. | Usually populated at runtime, otherwise empty |
| `work/` | Temporary overlayfs work directory. | Usually empty |
| `compat_base/`, `compat_base_overlay/`, `compat_base_overlay_upper/`, `compat_base_overlay_work/` | Compatibility and overlay backing trees used while preparing the runtime environment. | Overlayfs metadata, registry data, and scratch files |
| `build/`, `Desktop-Debug/` | CMake/Qt build output directories. | Generated binaries, object files, and build metadata |
| `BepInEx_win_x64_5.4.23.5/` | Unpacked BepInEx runtime bundle kept locally for setup/testing. | BepInEx distribution files |

## Notes

- The folders above are local-only workspace state and are not meant to be committed.
- The actual application code is in `src/`, with the UI and database code under `src/frontend/` and `src/datastore/`.
- `Game/` and `mod/` are intentionally described here because they contain non-source files that are useful during setup but should stay out of version control.
